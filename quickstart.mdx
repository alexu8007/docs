# Get Started with Polarity

Welcome to Polarity! This comprehensive quickstart guide will walk you through our 6-step onboarding process, designed to get you and your team up and running with AI-powered code optimization in just a few minutes.

<Note>
  **Dashboard-First Platform**: Polarity is entirely dashboard-based with no API or SDK setup required. Simply complete the onboarding flow and start optimizing your code immediately.
</Note>

## Prerequisites

Before you begin, ensure you have:

- **GitHub Account**: Active GitHub account with repository access
- **Repository Permissions**: Admin access to repositories you want to optimize
- **Team Information**: If setting up for a team, gather team details and member email addresses

<Warning>
  **GitHub App Installation Required**: You'll need to install the Polarity GitHub App during onboarding to enable repository monitoring and optimization features.
</Warning>

<Step title="GitHub App Installation & Repository Connection">
**Connect your repositories through secure GitHub App integration**

<Warning>
  **Critical Step**: This step is required for Polarity to monitor and optimize your repositories. Without the GitHub App installation, optimization features will not be available.
</Warning>

**GitHub App Installation Process**:

1. **Permission Configuration**: Review and grant necessary permissions
   - Repository access (read/write for optimization PRs)
   - Commit status access (for integration with CI/CD)
   - Pull request management (for automated reviews)
   - Issue management (for tracking optimizations)

2. **Repository Selection**: Choose repositories to monitor
   - **Individual Repositories**: Select specific repositories
   - **Organization-wide Access**: Grant access to all current and future repositories
   - **Selective Access**: Choose repositories by team, project, or language

3. **Initial Repository Analysis**: Automatic baseline establishment
   - Code quality assessment
   - Security vulnerability scanning
   - Performance analysis baseline
   - Optimization opportunity identification

**Verification & Testing**:
- **Connection Test**: Verify successful GitHub integration
- **Permission Validation**: Ensure all necessary permissions are granted
- **Initial Data Sync**: Confirm repository data is properly imported
- **Webhook Configuration**: Set up real-time event processing

</Step>

</Steps>

## Troubleshooting Common Issues

<AccordionGroup>
<Accordion title="GitHub App Installation Issues">
**Common Problems**:
- Insufficient repository permissions
- Organization restrictions on third-party apps
- Network connectivity issues during installation

**Solutions**:
- Verify you have admin access to target repositories
- Check organization settings for third-party app permissions
- Contact organization admin if installation is blocked
- Try installation from a different network if connectivity issues persist
</Accordion>

<Accordion title="Repository Access Problems">
**Common Problems**:
- Repositories not appearing in selection list
- Permission denied errors during initial sync
- Private repositories not accessible

**Solutions**:
- Ensure GitHub App has access to target repositories
- Verify repository visibility settings
- Check if repositories are part of organizations with restrictions
- Re-authenticate GitHub connection if access issues persist
</Accordion>

<Accordion title="Team Setup Challenges">
**Common Problems**:
- Team member invitation failures
- Billing configuration issues
- Permission assignment problems

**Solutions**:
- Verify email addresses for team invitations
- Check spam/junk folders for invitation emails
- Ensure billing information is complete and valid
- Review team role assignments and permissions
</Accordion>
</AccordionGroup>
