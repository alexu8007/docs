# /dashboard/pr - Pull Requests List

## Visual Components

### Page Header
**What you see**: Clean header with title and controls
- **Title**: "Pull Requests" in large text
- **Subtitle**: "Manage and review pull requests across your repositories" in gray
- **Repository count**: Shows "X repositories" on the right
- **Refresh button**: Circular arrow icon to reload PR list

### Stack Prompt Banner
**What you see**: Blue promotional banner below header
- **Icon**: Git pull request icon in a white circle
- **Title**: "Create stacked pull requests"
- **Description**: "Stack dependent pull requests for better code review workflow"
- **Button**: "Learn More" button on the right (blue)

### PR Category Groups
The main content is organized into collapsible groups. Each group has:

#### Group Header Bar
**What you see**: Clickable gray bar for each category
- **Expand/Collapse arrow**: Chevron pointing right (collapsed) or down (expanded)
- **Category icon**: Small icon indicating the type
- **Count and title**: "3 Needs your review" format
- **Item count**: "X items" on the right side
- **Hover effect**: Slightly darker background when mouse over
- **Click action**: Expands or collapses the group

#### Categories (In Order)

1. **Needs your review** (Red alert icon)
   - **When shown**: You're requested as reviewer
   - **Default state**: Expanded
   - **Priority**: Highest - these need immediate attention

2. **Returned to you** (Left arrow icon)
   - **When shown**: Your PRs with new feedback
   - **Default state**: Collapsed
   - **Priority**: High - you need to address comments

3. **Approved** (Green checkmark icon)
   - **When shown**: PRs approved but not merged
   - **Default state**: Collapsed
   - **Priority**: Medium - ready to merge

4. **Waiting for reviewers** (Users icon)
   - **When shown**: PRs without assigned reviewers
   - **Default state**: Collapsed
   - **Priority**: Medium

5. **Drafts** (Clock icon)
   - **When shown**: Draft PRs still being worked on
   - **Default state**: Collapsed
   - **Priority**: Low

6. **Merging and recently merged** (Git icon)
   - **When shown**: PRs in merge process or recently completed
   - **Default state**: Collapsed
   - **Priority**: Low - informational only

7. **Waiting for author** (Hourglass icon)
   - **When shown**: PRs blocked waiting for author
   - **Default state**: Collapsed
   - **Priority**: Low

### PR Table (Inside Each Expanded Group)
**What you see**: Clean data table with PR information

#### Table Headers
- **Pull Request** - PR details
- **Status** - Visual status indicator
- **Reviews** - Comment count
- **Checks** - CI/CD status
- **Changes** - Lines added/removed
- **Updated** - Last activity time

#### Table Rows
Each PR row shows:

1. **Pull Request Column**:
   - **Author avatar**: Small circular profile picture
   - **PR title**: Bold text, main title
   - **PR number**: Gray "#123" after title
   - **Repository**: "owner/repo" in gray below title
   - **Draft badge**: Gray "Draft" label if applicable

2. **Status Column**:
   - **Green check**: Ready/approved
   - **Purple check**: Merged
   - **Red X**: Has conflicts
   - **Yellow clock**: Pending/draft
   - **Gray circle**: Neutral

3. **Reviews Column**:
   - **Number badge**: Gray box with comment count

4. **Checks Column**:
   - **Green check**: All checks passing
   - **Red X**: Some checks failing
   - **Yellow clock**: Checks pending

5. **Changes Column**:
   - **Format**: "+123 / -45" in monospace font
   - **Colors**: Green for additions, red for deletions

6. **Updated Column**:
   - **Relative time**: "2 hours" (no "ago")

#### Row Interactions
- **Hover**: Light gray background appears
- **Click**: Navigates to PR detail page
- **Mouse hover**: Prefetches PR data for instant loading

### Empty States

#### No PRs At All
**What you see**: Centered message with icon
- **Icon**: Large git pull request icon
- **Title**: "No Pull Requests Found"
- **Message**: "Make sure you have repositories added to your dashboard and that they have open pull requests."
- **Button**: "Go to Dashboard" to add repositories

#### Empty Category
**What you see**: Category shows but table is hidden when count is 0

### Loading States

#### Initial Load
**What you see**: While PRs are loading
- **Spinner**: Animated circle in center
- **Progressive loading**: Page renders immediately, PRs appear as they load
- **Smooth appearance**: PRs fade in as they arrive

#### Refresh
**What you see**: When clicking refresh button
- **Button spins**: Refresh icon rotates
- **Existing data remains**: No flash or disappear

### Error State
**What you see**: When loading fails
- **Error message**: Red text explaining the problem
- **Retry button**: "Retry" with refresh icon
- **Centered layout**: In middle of page

### Visual Hierarchy
**How information is prioritized**:
1. **Needs review** group always expanded first
2. **Counts** prominently displayed in group headers
3. **PR titles** are the largest text in rows
4. **Status indicators** use color for quick scanning
5. **Metadata** (repo, time) in smaller gray text

### Responsive Design

#### Mobile View
- Groups stack vertically
- Table scrolls horizontally
- Reduced columns shown
- Touch-friendly tap targets

#### Desktop View
- Full table width utilized
- All columns visible
- Hover effects enabled
- Keyboard navigation supported

### Performance Features

#### Streaming Updates
**What you see**: PRs appear progressively
- No waiting for all PRs to load
- Smooth animation as PRs arrive
- Categories update counts in real-time

#### Prefetching
**What happens**: Hovering over a PR row
- Data loads in background
- Click = instant navigation
- No loading screen on PR detail page

### Color Coding System
- **Green**: Good/passing/approved
- **Red**: Needs attention/failing
- **Yellow**: Pending/waiting
- **Purple**: Merged/completed
- **Gray**: Neutral/draft/inactive
- **Blue**: Interactive elements/CTAs

### Interactive Elements

#### Clickable Items
- Group headers (expand/collapse)
- PR rows (navigate to detail)
- Refresh button (reload data)
- "Learn More" button (stack info)
- "Go to Dashboard" (when empty)

#### Hover Effects
- Group headers darken
- Table rows highlight
- Buttons change shade
- Links show underline

### Data Freshness Indicators
- **Updated time**: Shows age of each PR
- **Refresh button**: Manual update option
- **Auto-refresh**: On window focus
- **Repository count**: Confirms scope

# /dashboard/pr/[owner]/[repo]/[number] - Pull Request Detail View

## Page Layout
The page uses a three-column layout:
- **Left sidebar**: File list and navigation
- **Center**: Code diffs and main content
- **Right sidebar**: Checks, reviews, and AI tools

## Visual Components

### Top Title Bar (PRCompactTitleBar)
**What you see**: Compact header with PR information
- **Repository path**: "owner/repo" with folder icon
- **PR title**: Editable text (click to edit inline)
- **PR number**: "#123" in gray
- **Status badges**:
  - "Draft" (gray) if draft PR
  - "Merged" (purple) if merged
  - "Closed" (red) if closed
- **Action buttons** (right side):
  - "Approve" (green) - Submit approval
  - "Request Changes" (red) - Request modifications
  - "Merge" (blue) - When ready to merge
  - "Close PR" - Close without merging

### Tab Navigation
**What you see**: Two tabs below title
- **Files tab** (default): Shows code changes
- **Stack tab**: Shows related PRs (coming soon)

### Left Sidebar (File Navigation)

#### File Search Box
- **Search input**: "Filter files..." placeholder
- **Instant filtering**: Types to filter file list

#### File Tree
**What you see**: Hierarchical list of changed files
- **File icons**: Based on file type
- **File names**: Click to jump to that file
- **Change indicators**:
  - Green "+X" for lines added
  - Red "-X" for lines removed
  - Yellow "M" for modified files
  - Green "A" for added files
  - Red "D" for deleted files
- **Folder structure**: Collapsible folders
- **Current file**: Highlighted in blue

#### File Categories
Files are grouped by change type:
- **Modified** section
- **Added** section
- **Deleted** section
- **Renamed** section

#### PR Metadata (Bottom of Sidebar)
- **Author**: Avatar and username
- **Created**: Date/time
- **Updated**: Last activity
- **Base**: Target branch name
- **Head**: Source branch name
- **Commits**: Number with expandable list

### Center Content Area (Code Diffs)

#### Large PR Warning
**When shown**: PRs with >1000 lines or >10 files
- **Message**: "Large PR detected. Loading first 3 files..."
- **Load button**: "Load More Files" appears at bottom

#### File Diff Components
For each changed file:

##### File Header
- **File path**: Full path with folder structure
- **Action buttons** (right side):
  - "Viewed" checkbox - Mark file as reviewed
  - "Copy" icon - Copy file contents
  - "Collapse" arrow - Hide/show diff
- **Change summary**: "+50 -10" in green/red

##### Diff Viewer
**View modes** (toggle in top-right):
- **Unified view**: Single column with +/- indicators
- **Split view**: Side-by-side comparison

**What you see in diffs**:
- **Line numbers**: Clickable for permalinks
- **Syntax highlighting**: Language-specific colors
- **Added lines**: Green background with "+" prefix
- **Removed lines**: Red background with "-" prefix
- **Context lines**: Gray, unchanged code
- **Expand buttons**: "..." to show hidden context

##### Inline Comments
**What you see**: Comments on specific lines
- **Comment thread**: Nested conversation
- **Author avatar**: Small profile picture
- **Author name**: Bold text
- **Timestamp**: "2 hours ago" format
- **Comment text**: Markdown supported
- **Actions**:
  - "Reply" - Add to thread
  - "Resolve" - Mark as addressed
  - "React" - Emoji reactions
- **Add comment**: "+" button on line hover

#### Load More Section
**For large PRs**:
- **Progress indicator**: "Showing 3 of 15 files"
- **Load button**: "Load 5 More Files"
- **Auto-load**: Triggers when scrolling near bottom

### Right Sidebar Components

#### Checks Section (PRRightSidebar)
**What you see**: CI/CD and status checks

##### When checks exist:
- **Section title**: "Checks"
- **Check items**:
  - Check name (e.g., "Build", "Tests")
  - Status icon:
    - ✅ Green check - Passed
    - ❌ Red X - Failed
    - ⏱️ Yellow clock - Running
    - ⊖ Gray dash - Neutral
  - Duration time (e.g., "2m 30s")
  - "Details" link - View logs

##### When no checks:
- **Message**: "No checks configured for this repository"
- **Gray background**: Subtle indicator

#### Reviews Section
**What you see**: Review status and history

##### Review Summary
- **Required reviews**: "2 of 2 approved"
- **Review list**:
  - Reviewer avatar
  - Reviewer name
  - Review status:
    - ✅ "Approved" (green)
    - 💬 "Commented" (gray)
    - ❌ "Changes requested" (red)
    - 👁️ "Pending" (yellow)
  - Timestamp

##### Reviewer Management
- **Add reviewer button**: "+" icon
- **Search modal**: Find and add reviewers
- **Remove option**: "X" on hover
- **Re-request**: Click name to re-request

#### Assignees Section
**What you see**: PR assignees

- **Current assignees**: Avatar + name list
- **Add assignee**: "+" button opens search
- **Self-assign**: "Assign yourself" quick link
- **Remove**: "X" appears on hover

#### AI Review Panel (Paragon Integration)
**What you see**: AI-powered review section

- **"Run AI Review" button**: Purple gradient button
- **Review in progress**: Loading animation
- **Review complete**:
  - Overall score (e.g., "85/100")
  - Key findings list
  - Suggestions with severity:
    - 🔴 Critical
    - 🟡 Warning
    - 🔵 Info
  - "View Details" expandable sections

#### Labels Section
- **Current labels**: Colored badges
- **Add label**: "+" button
- **Label picker**: Dropdown with colors

#### Test Files Indicator
**When test files detected**:
- **Badge**: "3 test files"
- **Icon**: Test tube icon
- **Click action**: Opens test modal

### Modals and Overlays

#### Edit Description Modal
**Opens when**: Click PR title to edit
- **Text area**: Markdown editor
- **Preview tab**: See formatted result
- **Save/Cancel buttons**: Bottom of modal

#### Test Modal
**Opens when**: Click test files badge
- **Test file list**: With run status
- **Test results**: Pass/fail indicators
- **Coverage report**: If available

#### Add Reviewer Modal
- **Search box**: Find users
- **User list**: With avatars
- **Multiple select**: Checkboxes
- **Add button**: Confirms selection

### Visual States

#### Loading States
- **Initial load**: Skeleton placeholders for content
- **File loading**: Shimmer effect on file diffs
- **Lazy loading**: Progressive file appearance

#### Viewed Files
- **Unviewed**: Normal appearance
- **Viewed**:
  - Checkmark in "Viewed" button
  - Slightly faded appearance
  - Green checkmark in file list

#### File States in Diff
- **Collapsed**: Only header visible
- **Expanded**: Full diff shown
- **Loading**: Skeleton loader
- **Error**: Red error message

### Interactive Elements

#### Hover Effects
- **Line numbers**: Highlight and show "+" for comments
- **Buttons**: Darker shade
- **File names**: Underline in sidebar
- **Code lines**: Light highlight

#### Click Actions
- **File in sidebar**: Scrolls to file
- **Line number**: Creates permalink
- **Viewed button**: Toggles viewed state
- **Collapse arrow**: Hides/shows file
- **PR title**: Enter edit mode

### Performance Optimizations

#### Progressive Loading
**For large PRs (>1000 lines or >10 files)**:
1. Initial load shows first 3 files
2. "Load More Files" button appears
3. Loads 5 files at a time
4. Auto-loads when scrolling near bottom
5. Smooth animation as files appear

#### Caching
- **3-minute cache**: PR data cached
- **Prefetch from list**: Hovering prefetches
- **Instant navigation**: No loading screen

### Color Coding
- **Green**: Additions, approvals, passing
- **Red**: Deletions, failures, errors
- **Yellow**: Warnings, pending, in-progress
- **Blue**: Links, primary actions
- **Purple**: Merged status, AI features
- **Gray**: Neutral, viewed, inactive

### Responsive Behavior

#### Mobile
- Sidebars become collapsible drawers
- Single column layout
- Touch-friendly buttons
- Swipe between files

#### Desktop
- Three-column layout
- Hover interactions
- Keyboard shortcuts
- Resizable sidebars
