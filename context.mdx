# /dashboard/context - Context and Rules Page

## Visual Components

### Page Header
**What you see**: Top section with icon and title
- **Icon**: Settings gear in gray square
- **Title**: "Context and Rules"
- **Description**: "Manage repository contexts and rules for Paragon AI code reviews"

### Action Bar
**What you see**: Search and action buttons row
- **Search bar** (left):
  - Placeholder: "Search contexts and rules..."
  - Magnifying glass icon
  - Real-time filtering as you type
- **Buttons** (right):
  - "Add Context" (blue with + icon) - Primary action
  - "Custom Rule" (outline with code icon) - Secondary

### Rules Table
**What you see**: Main content table

#### Table Headers
- **Rule/Repository** - Repository and rule info
- **Created** - Date created
- **Type** - Guided or Custom
- **File Patterns** - Target files
- **Actions** - Manage options

#### Table Rows
Each row displays:
- **Repository name**: Bold text
- **Rule description**: Gray text below repo name
- **Type badge**:
  - "Guided" (gray with lightbulb icon)
  - "Custom" (blue)
- **Created date**: MM/DD/YYYY format
- **File patterns**:
  - Shows first 2 patterns as badges (e.g., "*.js", "src/**")
  - "+3" badge if more patterns
  - "All files" if no patterns specified
- **Status**: Green "Active" badge
- **Delete button**: Red trash icon

#### Empty State
**When no rules exist**:
- **Icon**: Large lightbulb icon (dimmed)
- **Message**: "No contexts found"
- **Subtext**: "Click 'Add Context' to get started"

### Table Footer
- **Left side**: "Total Contexts: X"
- **Right side**: "Refresh" button with icon

## Modal 2: Custom Rule Creation

### Modal Header
- **Title**: "Add Custom Rule"
- **Description**: "Create specific coding rules and guidelines for your repository."

### Repository Selection
(Same as Guided Context)

### Custom Rule Section
- **Label**: "Custom Rule *"
- **Textarea**: 5 rows
- **Placeholder**: "e.g., Always use async/await instead of .then() promises for better readability and error handling"
- **Helper text**: "Be specific about what should be done, avoided, or how code should be structured."

### File Patterns Section
**Header row**:
- **Label**: "File Patterns"
- **Subtext**: "Specify which files this rule applies to (optional)"
- **Add Pattern button**: Small with + icon

**Pattern inputs**:
- Multiple text fields (one per pattern)
- Placeholder: "e.g., *.ts, src/**/*.js, components/*.tsx"
- X button to remove (if more than 1)

**Info box** (gray background):
- **Title**: "Pattern Examples:"
- **Examples list**:
  - `*.js` - All JavaScript files
  - `src/**/*.ts` - All TypeScript files in src directory
  - `components/*.tsx` - All TSX files in components folder
  - `**/*.test.js` - All test files

### Modal Footer
- **Cancel button**: Gray outline
- **Save button**: "Save Custom Rule" (blue)

## Visual States

### Loading
- **Table**: Shows spinner with "Loading contexts..."
- **Modals**: Button spinner when saving

### Hover Effects
- **Table rows**: Light gray background
- **Buttons**: Darker shade
- **Delete icon**: Becomes darker red

### Form Validation
- **Required fields**: Red asterisk (*)
- **Error state**: Red border on invalid fields
- **Error message**: Red text below field
- **Success**: Green checkmark after save

### Dropdown Behavior
- **Closed**: Shows selected value or placeholder
- **Open**: Dropdown panel with options
- **Selected**: Blue background on current option
- **Hover**: Light blue on option
- **Custom selected**: Shows additional input field

## Color Coding
- **Blue**: Primary actions, selected items
- **Gray**: Secondary elements, guided type
- **Green**: Active status, success
- **Red**: Delete actions, errors
- **Yellow/Orange**: Warnings (not used here)

## Responsive Design

### Mobile
- **Table**: Becomes card layout
- **Modals**: Full screen
- **Grid**: Single column
- **Buttons**: Full width

### Tablet
- **Table**: Horizontal scroll
- **Modals**: Centered with margin
- **Grid**: Maintains 2 columns

### Desktop
- **Table**: Full width
- **Modals**: Centered, 60% width
- **Grid**: 2 columns as designed

## Interactive Elements

### Clickable Items
- Table rows (view details)
- Add Context button (opens modal)
- Custom Rule button (opens modal)
- Delete icon (removes rule)
- Refresh button (reloads data)
- Dropdowns (show options)
- Save/Cancel (modal actions)

### Real-time Features
- Search filters instantly
- Form validation on blur
- Success toast after save
- Table updates after actions

## Tooltips
**Hover to see**:
- File pattern badges show full pattern
- Truncated text shows full content
- Delete button shows "Delete rule"
- Type badges explain guided vs custom
